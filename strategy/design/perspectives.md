# 설계 관점 목록

설계 문서 작성 시 누락되기 쉬운 관점. 실전에서 "설계서엔 X라 했는데 구현하니 Y가 필요했다"는 경험에서 축적된다.

## 기본 관점

### 1. 에러 구체성 — 에러 흐름을 구체적으로 정의했는가
- 에러코드가 enum으로 정의되어 있는가 ("에러 반환" 이 아닌 "ErrorCode.TOKEN_NOT_FOUND, 404")
- 각 실패 지점에서 응답 형태가 명확한가

### 2. 타입 완전성 — 모든 인터페이스가 코드블록으로 정의되었는가
- 요청/응답 타입이 자연어가 아닌 코드로 정의되어 있는가
- optional 필드가 명시되어 있는가

### 3. 경계 조건 — 입력의 극단값을 고려했는가
- 빈 배열, null, 0, 빈 문자열일 때의 동작이 정의되어 있는가
- 페이지네이션의 첫 페이지와 마지막 페이지
