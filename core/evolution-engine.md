# Evolution Engine — 진화 스킬 공통 프로세스

이 문서는 실행 코드가 아니라 Claude가 진화 스킬 실행 시 참조하는 지침이다.
정적 스킬은 이 프로세스를 참조하지 않는다.

## 적용 대상

진화 스킬만 이 프로세스를 따른다. 진화 스킬은 실전에서 발견된 패턴을 strategy 파일에 축적하는 스킬이다.
현재 검증된 진화 스킬: test-evolve (테스트 도메인)

## 6단계 프로세스

### 1. DETECT (감지)

트리거 이벤트를 식별한다.

- 트리거 예시: 버그 수정 완료, 장애 발생, 리뷰에서 누락 발견, 배포 실패 등
- 입력: git diff, 에러 로그, 사용자 설명, 최근 fix: 커밋
- 이 단계의 목표: "무슨 일이 일어났는가?"를 명확히 정의

### 2. ANALYZE (분석)

근본 원인과 전략 갭을 분석한다.

- 근본 원인 파악: 이 문제가 왜 발생했는가?
- 기존 전략 검토: 해당 도메인의 strategy 파일(strategy/[domain]/*.md)을 읽는다
- 갭 분석: 기존 전략이 이 문제를 왜 잡지 못했는가?
  - 기존 관점/패턴에 속하지만 커버리지가 부족했는가?
  - 완전히 새로운 관점/패턴인가?
- 이 단계의 목표: "기존 전략의 어떤 부분이 부족했는가?"를 특정

### 3. EXTRACT (추출)

발견된 패턴을 구조화한다.

- 패턴 이름: 재사용 가능한 한 줄 이름 (예: "BigInt 직렬화 경계 검증")
- 구체적 항목: 이 패턴을 적용할 때 확인해야 할 것들
- 유래(provenance) 정보:
  - 프로젝트명
  - 날짜 (YYYY-MM-DD)
  - 구체적 사건 설명 (어떤 버그/장애에서 발견)
- 이 단계의 목표: strategy 파일에 기록할 수 있는 형태로 패턴을 정제

### 4. APPLY (적용) — 도메인 특화

발견된 패턴을 코드베이스에 구체적으로 적용한다.

- 이 단계는 도메인마다 다른 구체적 행동을 정의한다
- 각 스킬의 SKILL.md에서 이 단계를 도메인에 맞게 구체화해야 한다
- 예시:
  - 테스트 도메인: 누락된 테스트 작성/보강, 동일 패턴의 다른 모듈에도 테스트 추가
  - 리뷰 도메인: 코드리뷰 체크리스트에 항목 추가, 관련 코드에 주석/가드 추가
- 이 단계의 목표: 발견된 패턴이 실제 코드에 반영되는 것

### 5. RECORD (기록)

전략 파일과 프로젝트 CLAUDE.md를 업데이트한다.

- strategy/[domain]/*.md 파일에 추출된 패턴 추가
  - core/strategy-schema.md의 형식을 따른다
  - 유래 정보를 반드시 포함한다
- 프로젝트 CLAUDE.md의 관련 섹션 업데이트
  - 기존 관점이면: 구체적 항목 보강
  - 새 관점이면: 관점 추가 + 배경 한 줄 기록
- 이 단계의 목표: 발견된 지식이 영구적으로 축적되는 것

### 6. PROPAGATE (전파)

축적된 전략을 다른 프로젝트로 전파한다.

- 동일 패턴의 다른 모듈 보강 (프로젝트 전반은 사용자 요청 시에만)
- 플러그인 레포 동기화:
  - core/sync-rules.md의 규칙을 따른다
  - strategy 파일 변경 시 플러그인 레포에 커밋/푸시
- 이 단계의 목표: 한 프로젝트의 학습이 모든 프로젝트에 전파되는 것

## 스킬 작성 시 주의사항

- 6단계 중 Step 4 (APPLY)만 도메인별로 구체화가 필수. 나머지는 이 문서의 지침을 그대로 따른다.
- 새 진화 스킬 작성 시: templates/skill-template.md의 진화 스킬 템플릿을 복사하여 시작한다.
- 정적 스킬은 이 문서를 참조하지 않는다. templates/skill-template.md의 정적 스킬 템플릿을 사용한다.
